Lesson 1: Introduction to Networks

Network

A network is a group of entities that can communicate, even though they are not all directly connected. These entities may be people, computers or other objects which are connected via an intermediary.

Latency

Latency is the time it takes for a message to get from the source to its destination. This time is measured from the moment the message is started sending until it's received. Latency may be measured in milliseconds.

Bandwidth

Bandwidth is the amount of information that can be transmitted in a given amount of time - the rate of transfer. Bandwith may be measured in bits per second.

What is a bit?

A bit is the smallest unit of information. This takes the form of 0 and 1 (which represents a choice, for example yes/no or on/off).

Protocols

A protocol is a set of rules that allow two entities to communicate with each other. In the web this is the communication between the client (the web browser) and the server (e.g. udacity.com) and this takes place using the Hypertext Transfer Protocol (HTTP).

Lesson 2: Make the Internet Work for You

URLs

URL is an acronymn for Uniform Resource Locator. The URL is made up of three sections:
http --> Protocol
://
www.udacity.com Host (this can also be an IP address)
/ --> Path

Query Parameters (GET Parameters)

http://example.com/foo?p=1
Name = value
To add extra parameters a ? is used.
http://example.com/foo?p=1&q=neat

Fragments

A fragment is generally used to reference a particular part of the page. The fragment is not sent to the server when making a request purely used in the browser. E.g.
http://www.example.com/foo#fragment
http://example.com/foo?p=1#fragment

Ports

An internet connection requires a host and a port. By default the port is 80. However, an alternative port may be accessed by including the port in the path in between the host and the path separated by a colon. E.g.
http://localhost:8000/

HTTP Request

Hyper Text Transfer Protocol works by sending a request line. This is divided into 3 sections: method, path and version.
http://www.example.com/foo
GET /foo HTTP/1.1 --> request line
GET --> method - what type of request is being made to the server
foo --> path - the document requested from the server
HTTP/1.1 - version (either 1.1 or 1.0)
Note: The host name is not a part of the request line in the relative path for that URL.
After the request line there are a number of headers.
Host: www.example.com
User-Agent: chrome v.17

HTTP Responses

A reponse is in the form of the status line.
HTTP/1.1 200 Ok
HTTP/1.1 --> version - this matches the version of the request line
200 --> status code
Ok --> Response phrase
Other common status codes: 302 Found, 404 Not found, 500 Server Error
The status line is also followed by a number of headers. E.g.
HTTP/1.1 00 Ok
Date: Tue Mar 2012 04:33:33 GMT --> when the request was sent
Server: Apache /2.2.3 --> the server handling the response
Content-Type: text/html;
Content-Length:1539

Servers

Purpose: repond to HTTP requests. There are two classifications of server responses: static and dynamic. Static content is pre-written files, examples include images. Dynamic responses are made on the fly by web applications (programs which generate content).

Lesson 3: Forms

HTML Forms

HTML forms use the tag <form>. Input may be added into the form tag, this may have an attribute called name with any value. E.g.

<form>
 <input name="name">
</form>

A submit button can be added to the input too by adding <input type="submit">

Action is an attribute which can be added to form and will send the result here. E.g. <form action="/foo">

The type of the input may be defined. By default the type is text, however this may be changed. E.g. <input type="password"> or <input type="checkbox"> or <input type="radio">

A value may be added to control the output for radio buttons or checkboxes. E.g. <input type="radio" name="q" value="one">

Label Element

The label element may be used to give a title to each specific input used on a form. E.g.

<form>
 <label>
  One
  <input type="radio" name="q" value="one"
 </label>
 <label>
  Two
  <input type="radio" name="q" value="two"
 </label>
</form>

Select Element

A select element is used to provide a drop-down list. E.g.

<form>
 <select name="q">
  <option>one</option>
  <option>two</option>
  <option>three</option>
 </select>
</form>

Lesson 4: Modules and Dictionaries

Modulus Operator

The modulus operator is the remmainder from divided a number into another number. E.g.

15 % 12 --> 3
15 % 11 --> 4
4 % 3 --> 1
3 % 4 --> 3

Dictionary

A dictionary is created using {}, ad the entries are key value pairs. The keys can be any immutable value. Dictionary is mutable. The value of the key may be displayed by using d[k] - where the value of k in d will be displayed.E.g.

elements = { 'hydrogen': 1, 'helium': 2, 'carbon': 6 }
print elements> --> this will print the dictionary
print elements['hydrogen'] --> this will print the value 1
print elements['lithium'] --> this will return a key error as there is no lithium in the dictionary
print 'lithium' in elements --> this will return false as lithium is not in the dictionary
elements['lithium'] = 3 --> this will add lithium to the dictionary

HTML Templates

Templates refer to a libary used to build complicated strings. In Google App Engine we can use junja2 which is built in. Templates allow programmers to avoid repetition which means complicated code is reduced and the room for errors is removed. In addition templates mean the applications or web pages created can be scalable and not require new code to be created each time there are any additions. The code may also be change in one place which will alter the whole process rather than needing to edit each separate addition of code which would be necessary without templates. Validation is crucial to ensure the security of applications preventing malicious attacks for example from forms on websites, by prohbiting the addition and execution of code.
